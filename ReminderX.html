Problem

ReminderX (a startup software company) has built and launched a to-do list reminder mobile application. ReminderX wanted to know which user groups to target and gain more insight from these users on their problems. ReminderX product management team wanted to determine what additional features needs to be added or changed based on these user requirements.

Actions (My Approach)

My approach to address the gap in their product strategy was the following actions:

User Interviews

The interviews from a sample of people help form the prototypical user and features desired.

Prototyping

Rapid prototyping on paper to enable user testing.

Usability Testing

User recruitment and testing with users using paper lo-fidelity prototypes.

Testing Analysis

Gathering, sorting, and analyzing user findings to determine design pivot opportunities and communicating these needs to ReminderX.

Iterate

Iterate prototype from user findings while adehering to earlier discovered design principles/tenets.

Results

This User Experience Design stepwise approach allowed me to distill the right insight to provide a fast prototype that addresses user needs and provides ReminderX the product roadmap critical to their product line.

Lessons Learned

This experience has allowed me to implement fast paper prototyping testing. I was able to determine that iteration is necessary as the first design proposal has many improvement opportunities. However, the design principles and tenets still need to hold true.

Design Tenets

  • Make It Familiar
  • Make It Simple
  • Make It Remind
  • Make it Work with My Current Tools
  • Make It Accessible Anywhere
  • Make It Collaborative
